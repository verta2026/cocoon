# 给你的 agent 看的：住在 cocoon 里

*这份文档的读者是 agent 本身——跑在 cocoon 里的那个 Claude Code。部署者：
在 `CLAUDE.md` 里指一句让它来读（文末有现成的片段）。以下所有内容直接对
agent 说话。*

---

你正运行在 cocoon 里。你的终端会话被渲染成了浏览器里的聊天页——多半在
一台手机上。你的工作方式没有任何变化：工具、hooks、记忆、slash 命令照常。
变的是你的话**长什么样**，以及有几个文本标记能解锁前端功能。这一页告诉你
屏幕那头的人真正看到了什么。

## 你的输出会变成什么

纯文本变成聊天气泡。支持一个 markdown 子集：

- **粗体**、*斜体*、`行内代码`、标题、列表、`> 引用`、分隔线、链接
- 围栏代码块——渲染后自带复制按钮，所以任何用户可能要粘贴的东西
  （命令、配置、prompt）都放进围栏
- 表格（`|` 竖线语法）和盒线表都会渲染成真表格
- 你的工具调用（读文件、跑命令……）默认折叠，用户看到一行可展开的摘要

按手机屏幕来写：短段落好过宽表格，长输出放围栏别裸排。

## 你会收到什么

- 普通消息就是用户敲的原文。
- **引用回复**：用户左滑气泡回复时，你收到的消息以 `> <被引用的那行>`
  开头，下一行是他们说的话。
- **图片上传**：以 `[图片] <路径>` 到达——那是本机上真实的文件路径，
  用你的 Read 工具打开就能看。上传的图你**看得见**，贴纸的图才看不见
  （见下）。
- **文件上传**：以 `[文件] <路径>` 到达，照常读。
- **贴纸**：以 `[sticker:<文件名>|<名字>|<描述>]` 到达——前端在用户点选时
  就把 `meta.json` 里的名字和描述嵌进了标记，你不用看图就知道对方发的是什么
  心情。收到不带竖线的裸 `[sticker:<文件名>]` 说明那张还没写元数据。

## 你怎么往回发

- **图片**：写 markdown 图语法——上传目录里的文件用
  `![](/files/<文件名>)`，或任何 `https://` 图片直链、`data:image/...`。
  鉴权由前端自动补，**永远不要自己往 URL 里拼 token**。
- **文件**：提路径即可，`/files/` 下的会渲染成可下载的附件条。
- **贴纸**：单独写一行 `[sticker:<文件名>]`。发之前先读贴纸目录的
  `meta.json` 挑——`name`/`desc` 字段就是你认识每张贴纸的方式。你永远
  看不见贴纸图片本身，描述就是你的眼睛。不要编造文件名。
  （完整协议：README → 贴纸。）
- **语音**（仅当这个部署配了 TTS）：`[[voice:<id>]]` 标记会渲染成可播放
  的语音气泡。音频怎么生成是部署各自的事——如果你不知道自己部署的 TTS
  是怎么配的，那就是没有，别发这个标记。

## 向用户提问

如果装了弹窗 hooks（README → AskUserQuestion），你的 AskUserQuestion
工具调用会渲染成原生感的弹窗：按钮、多选、自定义文字——用户点选代替打字。
两条礼节：

- 不要把"先做 X 再回答"的前置说明和提问放在同一轮——弹窗会盖住你的
  消息，用户没读到说明就先点了。
- 用户直接打字回答时，弹窗会替他们收掉；两条路最后都回到你的会话里。

## 你永远看不见的东西

表情反应、气泡长按菜单、换壁纸换头像、日夜切换——全是纯前端行为，
不要等它们、也不要问。另外用户可能从历史检索或折叠段里读你，
每条回答写成独立可读的。

粘进你项目的 `CLAUDE.md`（路径自己调整）：

```markdown
# 聊天前端
你运行在 cocoon 里，终端被渲染成手机聊天页。
先读一遍 <cocoon 仓库>/docs/for-your-agent.zh-CN.md 了解消息标记
（贴纸、图片、引用、弹窗）。贴纸索引：<贴纸目录>/meta.json。
```
